## Parse coverage methodology

Fixture parse coverage is a CI canary: a fixture counts as fully parsed only when parsing succeeds with zero dropped events and zero unknown top-level fields.
Each built-in adapter currently has one conformance fixture, so `1/1 (100%)` does not claim 100% production reliability. Live workspace coverage in Settings is measured across actual changed-stream parse attempts.

## What “estimated” means

- **Measured** — token counts reported by the agent runtime or OTLP ingest.
- **Estimated** — derived from heuristics (chars/4, tiktoken when `precise` extra installed, model-specific tables).
- **Absent** — no usage data; cost may still be priced from model defaults with explicit `cost_source`.

Verdicts in Optimize use anytime-valid confidence sequences with clustered effective sample sizes — not fixed-width CIs under continuous monitoring.
