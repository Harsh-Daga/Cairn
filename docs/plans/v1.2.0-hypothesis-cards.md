## EXP-I — Intent Contract & requirement coverage

| Field | Content |
| --- | --- |
| Hypothesis | Explicit per-session intent + requirement checkboxes improve correction quality and reduce false "done" claims. |
| User / job | Operator verifying an agent finished the asked work. |
| Success metric | Precision/recall of requirement coverage vs human labels on ≥50 seeded sessions. |
| Failure mode | Invented requirements; UI implying enforcement. |
| Non-claims | Not a product requirement tracker; not CI gating without labels. |
| Flag | `experimental.intent_contract` (off by default) when implemented. |
| 1.2.0 decision | **Defer** — no production UI/API. |
