work remains. A tool timeout, context limit, or long test run is not completion; record state and
resume.

affected tracker rows, and smallest external action needed. External GitHub/PyPI settings do not
block completing the local implementation and readiness checklist.

You may declare completion only when:

- every master-prompt requirement is implemented, explicitly validated, or handled exactly as the
  master allows;
- every tracker row has an evidence-backed terminal state and nothing was silently deferred;
- the complete local release gate and required acceptance matrix pass;
- generated files and docs are current;
- the required final report from the master prompt is delivered with real results, remaining
  external maintainer actions, and unresolved risks;
- `git status` is inspected and you confirm that no commit, push, tag, publish, release, or remote
  configuration occurred.
