# Cairn 1.2.0 audit

Date: 2026-07-17  
Branch observed: `main` at the local `origin/main` tip  
Canonical specification: [`v1.2.0-revamp-prompt.md`](v1.2.0-revamp-prompt.md)

This audit records the verified starting point for the 1.2.0 implementation. It is evidence, not
a release claim. Measurements are local macOS observations unless a platform is stated.

## Executive finding

Cairn 1.1.1 is a coherent local trace-analysis product with strong foundations: strict Python
typing, a single-writer SQLite store, deterministic demo data, twelve built-in adapters, bounded
SSE client queues, action-registry mutations, evidence-bearing detectors, static export, MCP, and
an established measured-improvement loop. The baseline gates are mostly healthy.

The release is not yet safe to describe as the local verification and improvement control plane
specified for 1.2.0. The largest gaps are:

1. several global-history adapters can copy unrelated-project sessions into the active workspace;
2. static exports can contain raw trace content and an absolute workspace path;
3. public HTTP query parameters are not consistently bounded, and error shapes are inconsistent;
4. local trace files are created with group/world-readable modes on this machine;
5. the browser “Watch” control does not stop backend discovery/ingest work;
6. background jobs create an unbounded thread per submission;
7. no verification receipt, claim ledger, debt model, regression artifact, review queue, handoff,
   policy, archive, egress ledger, or Resource & Privacy Shield exists;
8. CI can report successful browser coverage while all browser tests are skipped;
9. release publishing is triggered by every `main` push and actions are not SHA-pinned;
10. the UI is dark-only and has material keyboard, reflow, and accessible-name gaps;
11. current API/UI/static/documentation contracts already disagree.

The implementation decision is therefore to preserve the proven ingest/analyzer/experiment
architecture, extend it with explicit verification and lifecycle domains, and replace unsafe or
misleading boundary behavior before broad visual work.

## Product and market inventory

Current official documentation confirms that generic observability products already combine
traces with sessions, evaluation, prompt management, datasets, experiments, replay, and human
annotations. Examples include [Langfuse sessions](https://langfuse.com/docs/observability/features/sessions)
and [Phoenix](https://arize.com/docs/phoenix). Local coding-agent products already provide
multi-agent usage reports, custom timezones and date filters
([ccusage](https://github.com/ryoppippi/ccusage)) and session replay, subagent trees, tool
inspection, and local token/cost tracking
([Agents Trail](https://camtrik.github.io/agent-trail/)). Benchmark viewers already combine
step replay with verifier output and pass rates
([Trajectories](https://www.trajectories.sh/)).

The specified positioning remains valid: Cairn's defensible category is coding-specific local
evidence, verification, recovery, and measured instruction improvement—not another waterfall or
cost dashboard.

The cited user signals were rechecked:

- Stack Overflow's 2025 survey reports 46% distrust AI-output accuracy
  ([source](https://survey.stackoverflow.co/2025/ai)).
- Sonar's 2026 survey reports 96% do not fully trust AI-generated code, 48% always verify before
  commit, and 38% report more review effort
  ([source](https://www.sonarsource.com/blog/state-of-code-developer-survey-report-the-current-reality-of-ai-coding/)).
- The 20,574-session misalignment paper describes seven recurring forms and says visible
  resolutions usually require explicit correction
  ([arXiv](https://arxiv.org/abs/2605.29442)).
- Holistic failure diagnosis argues that outcome-only evaluation does not localize failure and
  reports benefits from structured span-level analysis
  ([arXiv](https://arxiv.org/abs/2605.14865)).

These are directional third-party findings, not Cairn user research. No Cairn interviews or
five-user usability study were available in this repository, so feature demand remains a
hypothesis until locally collected, privacy-safe feedback exists.

### Personas, jobs, and current friction

| User/job | Current journey | Friction | Desired evidence | Local success signal |
|---|---|---|---|---|
| Solo power user: stop waste | Overview → Session → Context | Activity/cost, not verified progress | last verified progress, loop evidence, handoff | finds a concrete next action |
| Reviewer: decide trust | Session waterfall → raw spans | No claims/tests/edit ordering | receipt, debt, risky files, temporal tests | explicit accept/reject input |
| Maintainer: review AI change | Manual transcript/PR review | Raw transcript is private and huge | scrubbed manifest and receipt | privacy-safe export produced |
| Lead: compare workflows | Agents/Quality | No minimum samples/confounds | verified success, debt, correction burden | aggregate viewed above threshold |
| Harness author: debug | Session/Behavior | No reusable failure artifact | regression task and span diagnosis | failure round-trips |
| Privacy-sensitive user | Settings/doctor | Does not show actual footprint or modes | process/disk/permissions/egress facts | shield inspected/action completed |
| New user | Sync → chart wall | Empty states are generic | one finding, session, and next check | first useful evidence recorded |

No individual productivity ranking is recommended. Team evidence must be task-level, thresholded,
previewed, and explicitly exported.

Stable action names (16): `sync`, `backfill`, `rebuild_view`, `check`, `export_bundle`,

| Existing detectors | Overlap | Evidence/estimate state | Decision |
|---|---|---|---|

### Database and lifecycle inventory

Numbered append-only migrations currently run from `0001_init.sql` through
`0006_experiment_export_fields.sql`. Core tables cover workspaces, actors, traces, spans, links,
context regions, fingerprints/baselines, outcomes, diagnostics, evidence, view state, daily
rollups, insights/states, experiments, annotations, data quality, ingest cursors, file-read cache,
MCP consultations, and adapter parse health.

New verification, correction, regression, policy, Guard, egress, archive, resource, retention, and
job state need explicit numbered migrations or documented derived/rebuildable storage. Migration
backup/interruption/corruption/real pre-1.2 fixture coverage is absent. Doctor lacks
`PRAGMA integrity_check`.

Search currently pays for an FTS table seeded in demo data while production search uses `LIKE`;
transactional FTS lifecycle is not demonstrated.

Global-history Aider, OpenCode, Goose, Gemini, Cline-family, and OpenClaw discovery is not reliably
project-filtered. The active workspace database can therefore receive prompts/tool output from
other repositories and attribute them to the wrong workspace. Adapters must either prove a
workspace match, label a session as explicitly global/unattributed with user approval, or skip it;
silent cross-project copying is a P0 privacy and correctness defect.

### Background process inventory

| Process/work | Trigger/cadence | Bound today | Risk | Decision |
|---|---|---|---|---|
| DB writer thread | store open | queue max exists | shutdown/full disk diagnostics | keep/strengthen |
| watcher poll thread | UI pipeline start, 0.5 s | event queue bounded | historical file stat work | replace with modes/backoff |
| watcher event loop | UI pipeline start | queue bounded | duplicate callback/queue ingest | merge to one path |
| refresh thread | immediate + every 30 s | none for discovery work | repeated redetection/stat scans | adaptive efficient mode |
| action jobs | each async action | **thread per job** | unbounded threads/records | bounded executor |
| SSE clients | browser connection | per-client queue bounded | reconnect/history semantics | keep/extend |
| reflector HTTP | explicit optimize option | timeout exists | payload/provenance/egress absent | explicit preview + ledger |

The browser “Watch” toggle controls SSE invalidation, not backend auto-sync. Product labels are
therefore misleading.

## Accessibility and usability findings

Strengths: bundled local fonts, native buttons for waterfall rows, a visible focus style,
reduced-motion CSS, main/navigation landmarks, and waterfall virtualization.

Release blockers:

- dark-only `color-scheme`; no pre-mount light/system theme — resolved by T03-01 with a
  strict-CSP pre-mount bootstrap, migrated persisted preference, semantic light/dark tokens,
  contrast tests, and system-change browser coverage;
- collapsed navigation links lose their accessible names — resolved in T03-02;
- no skip link or forced-colors rules — resolved in T03-01/T03-02;
- mobile navigation exposes only five routes and covers page content — all routes are now exposed
  in a bounded horizontal dock and 320 px/200% reflow is browser-tested;
- command palette lacks focus trap/restore and `aria-modal` — resolved in T03-02 and shared by the
  shortcuts and custom-range dialogs;
- missing required keyboard shortcuts and `j/k` span traversal — session and waterfall traversal
  are implemented and browser-tested;
- chart inspection is often pointer-only and lacks text/table alternatives;
- no automated accessibility dependency or browser gate — component axe and all-route,
  both-theme Playwright axe gates now exist;
- 320 px/200% reflow and screen-reader smoke are undocumented/unverified — automated reflow passes
  and the manual screen-reader matrix is documented; an actual manual assistive-technology run
  remains required before final release acceptance.

## Security and supply-chain findings

- static payloads bypass the scrubber and expose raw detail/workspace paths;
- HTTP query bounds and large/deep payload limits are incomplete;
- auth exists for non-loopback binds, but Host/Origin/DNS-rebinding/CORS/header behavior lacks the
  required matrix;
- token bootstrap uses a query parameter before redirect; leakage/cache/referrer tests are absent;
- optional LLM trace-content separation and narrow output schemas are incomplete;
- global-history adapters can ingest unrelated-project private content;
- file/export/config permission defaults are permissive on the observed Unix host;
- all GitHub actions use movable tags;
- CI has broad workflow gaps, no CodeQL/Scorecard/dependency review/SBOM/provenance;
- publish runs on every `main` push;
- Dependabot version updates are configured, but security-update settings require a maintainer;
- package install tests ignore doctor failures on Unix and Windows;
- npm reports five audit findings that require triage, not an automatic breaking upgrade.

## Architecture decisions

The required ADRs live under `docs/architecture/decisions/` and are tracked separately. Audit-level
decisions:

- **keep** the single-writer store, adapters, action registry, analyzers, detector lifecycle,
  deterministic demo, stdio MCP, and static-data adapter;
- **extend** the existing trace/outcome/evidence models with verification and correction domains;
- **merge** overlapping detector families and Sessions/Search filter grammar;
- **replace** unbounded job threads, misleading Watch semantics, raw static payload export,
  handwritten transport duplication, and main-push publishing;
- **new** bounded resource modes, receipt/regression/archive schemas, egress ledger, evidence
  closure, light/system theme, Guard event storage, and privacy-safe lifecycle controls.

## Dependency proposal

No dependency is approved merely by this audit.

| Candidate | Need | Existing/platform alternative | Cost/posture | Removal path | Initial decision |
|---|---|---|---|---|---|
| ESLint + TS/React plugins | required lint gate | TypeScript cannot enforce hooks/style rules | dev-only, common ecosystem | remove config/scripts | add, pinned lock |
| `@vitest/coverage-v8` | UI branch baseline | V8 built into runtime but adapter needed | dev-only | remove reporter | add |
| `@axe-core/playwright` or `axe-core` | automated a11y | manual checks alone insufficient | dev-only, mature | isolate helper | add |
| OpenAPI TS generator | transport source of truth | small in-repo generator may suffice | dependency/bundle risk varies | committed generated output | prefer in-repo generator first |
| filesystem watcher | efficient cross-platform events | adaptive polling can satisfy initial contract | native maintenance risk | polling fallback | do not add yet |
| archive/compression library | safe streaming archives | stdlib zip/tar need hard limits | runtime attack surface | versioned adapter | prefer stdlib bounded ZIP initially |

Every added dependency must be re-evaluated against maintenance, advisories, lockfile impact, and
license notices at implementation time.

## Implementation order, risk, and rollback

1. **P0 truth and boundaries:** query limits/errors, static scrubbing, auth/header tests, private
   modes, project attribution, docs/security truth. Roll back additively; do not remove legacy
   shapes.
2. **Contracts:** time range, OpenAPI/generated types, configuration, migrations, archive schema.
   Preserve `days`, routes, fields, commands, actions, and existing DBs.
3. **Verification core:** receipt, ledger, debt, next evidence, review queue. Version calculations
   and keep deterministic rebuilds.
4. **Regression/handoff/corrections/policies:** all previewable and offline; mutations use actions.
5. **Resource/privacy:** bounded executor, explicit collection modes, retention/content controls,
   permissions, egress ledger, circuit breakers.
6. **Guard and product analytics:** extend existing experiment/detector evidence; label association.
7. **UI system and pages:** semantic tokens, accessible primitives, route chunks, new/extended
   pages, static read-only states.
8. **Docs/release automation:** generated contracts, hardened CI, tag build-once flow,
   provenance/SBOM/readiness evidence.
9. **Complete acceptance matrix:** clean installs, migrations, offline/no-network, hostile inputs,
   file exports, a11y, large fixtures, packaging, reproducibility.

The detailed durable state is [`v1.2.0-implementation.md`](v1.2.0-implementation.md). No item may
be marked complete without commands/evidence recorded there.
